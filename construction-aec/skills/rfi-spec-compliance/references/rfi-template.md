# RFI template & worked example

## Fields

| Field | Notes |
|---|---|
| RFI No. | Placeholder, e.g. `RFI-<disc>-<seq>` |
| Project / Contract No. | |
| Date raised | |
| Required by | State if time-sensitive; note programme impact |
| From (Contractor) | Company, name, role |
| To (Consultant/Engineer) | Company, name, role |
| Discipline | Architectural / Structural / MEP / Civil / Interfaces |
| Subject | One line |
| Reference documents | Drawing nos. + revision, spec section/clause, location (grid/level/room) |
| Question / Clarification sought | Factual, specific, answerable |
| Contractor's proposed solution | Optional but speeds response |
| Cost impact | Yes / No / TBC |
| Time impact | Yes / No / TBC |
| Attachments | Marked-up drawings, photos, datasheets |
| **Consultant response** | (blank block) |
| Responded by / date | (blank) |
| Status | Open / Answered / Closed |

## Writing guidance
- One question per RFI where practical — makes tracking and close-out clean.
- Cite exact drawing numbers **and revisions**; ambiguity often comes from working off superseded revisions.
- Frame around a discrepancy or missing information, not a design request.
- Offer a proposed resolution when you have one — consultants can often just endorse it.
- State cost/time impact honestly; flag if it is on the critical path.

## Worked example

**Subject:** Discrepancy in blockwork wall thickness at Level 2 corridor.

**Reference:** Architectural drawing A-201 Rev C shows 200 mm blockwork to corridor walls on grid C/3–C/6; Structural drawing S-110 Rev B notes 150 mm. Fire strategy F-002 requires a 2-hour rating to this corridor.

**Question:** Please confirm the required blockwork thickness to the Level 2 corridor walls (grids C/3–C/6), and confirm the confirmed thickness achieves the 2-hour fire rating per F-002.

**Contractor's proposed solution:** Adopt 200 mm blockwork per A-201 Rev C, subject to confirmation that this meets the 2-hour rating.

**Time impact:** Yes — blockwork to this zone is scheduled to start in 5 working days; a response is requested by <date> to avoid delay.
