# Submittal package — structure, status codes & worked example

## Transmittal fields

| Field | Notes |
|---|---|
| Submittal No. | Placeholder, e.g. `SUB-<spec-sec>-<seq>` or `MAR-<seq>` |
| Project / Contract No. | |
| Spec section | e.g. "09 30 00 — Tiling" |
| Revision | 0 for first issue; increment on resubmission |
| Date | |
| From (Contractor) | |
| To (Consultant/Engineer) | |
| Type | Material / Sample / Shop drawing / Prequalification |
| Description | Item + manufacturer + model |
| Action requested | Approve / Approve as noted / Review |
| Enclosures | Datasheet, certificates, samples, catalogue, warranty |
| **Consultant action** | Status code + comments + name/date (blank) |

## Consultant status codes (typical)
- **A — Approved / No objection**: proceed.
- **B — Approved as noted**: proceed incorporating comments; no resubmission needed.
- **C — Revise & resubmit**: address comments and resubmit; do not proceed.
- **D — Rejected**: not acceptable; submit an alternative.

(Confirm the exact code set from the project's document-control procedure — projects vary.)

## Compliance statement (clause-by-clause)

| # | Spec requirement | Clause | Proposed item value / evidence | Comply (Y/N/Comment) |
|---|---|---|---|---|
| 1 | Slip resistance R10 (wet areas) | 09 30 00 cl. 2.4 | Datasheet: R10 (DIN 51130) | Y |
| 2 | Water absorption ≤ 0.5% (porcelain) | cl. 2.2 | 0.05% (ISO 10545-3) | Y |
| 3 | Fire class | cl. 2.6 | Not stated on datasheet | Comment — cert to follow |

## Worked example — Porcelain floor tile (Spec 09 30 00)

**Transmittal:** "We enclose for your review and approval the material submittal for porcelain floor tiles (Manufacturer X, Model Y) proposed for Level 1–3 lobbies and corridors, in accordance with Specification Section 09 30 00. Enclosures: technical datasheet, ISO 10545 test certificates, slip-resistance certificate, and one A4 sample board. Action requested: Approve."

**Compliance statement:** clause-by-clause matrix as above, evidencing each spec requirement against the datasheet, with any gaps flagged for certificate follow-up.

**Consultant action block:** status code + comments + signature/date (left blank for the reviewer).

## Tips
- Order the package the way the reviewer checks it: transmittal → compliance statement → datasheets → certificates → samples schedule.
- Highlight where the item is an "approved equal" to a specified product, and evidence equivalence clause-by-clause.
- Track everything in a submittal register (spec section, submittal no., date submitted, status, date returned) — offer to build one as a spreadsheet.
- For resubmissions, clearly mark revision and show how prior comments were addressed.
