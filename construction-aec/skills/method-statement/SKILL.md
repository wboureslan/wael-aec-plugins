# Method Statement

Produce a professional construction method statement (MS) that a consultant or client can review and approve. A method statement describes HOW a specific activity will be executed safely and to spec.

## Step 1 — Gather the essentials

Identify the activity and its context before writing. If any of these are unknown and material to the statement, ask the user concisely (batch the questions):

- **Activity / scope** — e.g. internal blockwork, gypsum board partitions, screeding, waterproofing to wet areas, ceramic tiling, MEP first fix.
- **Project & location** — project name, building/area, floor/zone if relevant.
- **Reference documents** — applicable drawings, specification section, and any approved materials or shop drawings.
- **Standards / codes** — project spec clauses, and applicable codes (e.g. local building code, BS/EN, ASTM, ACI, local authority requirements). Default to the region implied by the project; if unclear, ask.

If the user already attached drawings, specs, or a scope document, read them first and pull the activity details from there instead of asking.

## Step 2 — Build the method statement

Structure the document with these sections. Keep it practical and specific to the activity — avoid generic filler.

1. **Document control** — title, project, document number placeholder, revision, date, prepared/checked/approved by (leave signature blocks).
2. **Purpose & scope** — what activity this MS covers and its boundaries.
3. **References** — drawings, specification sections, approved submittals, standards/codes.
4. **Definitions & abbreviations** — only if needed.
5. **Responsibilities** — project manager, site engineer, foreman, safety officer, QA/QC engineer.
6. **Resources**
   - **Manpower** — trades and roles.
   - **Plant & equipment** — tools and machinery.
   - **Materials** — with reference to approved material submittals.
7. **Sequence of works** — the core section. Numbered, chronological steps from preparation through execution to protection/curing. Be specific to the activity (e.g. for blockwork: setting out, dowel/starter bars, first course bedding, verticality checks, mortar mix, control joints, curing).
8. **Inspection & test plan (ITP) hooks** — hold points and inspection stages where the consultant is notified (reference to a WIR — Work Inspection Request).
9. **Health, safety & environment** — key hazards for this activity and their controls, PPE, and reference to the risk assessment. See `references/hse-hazards.md` for a hazard bank by activity.
10. **Quality control** — tolerances, acceptance criteria, and checks tied to the spec.
11. **Attachments** — risk assessment, ITP, approved drawings/materials (list as placeholders).

Refer to `references/ms-structure.md` for a fuller annotated outline and a worked example.

## Step 3 — Produce the deliverable

Default to a Word document unless the user asks otherwise. Use the `docx` skill to build a clean, formatted MS with a title block, numbered headings, and signature/approval blocks. Offer a PDF export if the user needs to transmit it for approval.

Leave clearly marked placeholders (document number, dates, names, drawing/spec references) where the user must fill in project-specific values — do not invent project numbers or personnel names.
